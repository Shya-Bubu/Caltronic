# Exam Focus - How to Build the Routh Array

## What is usually tested

This concept often appears as a procedural question:

- construct the Routh array for a given denominator
- identify first-column signs
- conclude whether the system is stable

## How to maximize marks

Examiners reward clean structure. Do this:

1. write the polynomial in descending powers of $s$
2. copy the first two rows carefully
3. compute each later row entry with the correct formula
4. box or underline the first column
5. write the final stability conclusion

## Common mistakes

- skipping missing coefficients in the polynomial
- copying the first two rows incorrectly
- dividing by the wrong first-column element
- forgetting the epsilon substitution when a denominator entry becomes zero
- building the array correctly but not counting sign changes

## One line you should remember

> The system is stable if and only if every entry in the first column of the Routh array is positive.
